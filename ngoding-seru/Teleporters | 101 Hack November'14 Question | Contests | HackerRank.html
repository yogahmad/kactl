    ByteLand is a large country with a tree-like road system. It has cities and unidirected roads. The capital of ByteLand is a city, numbered , and all the roads go away from the capital. So people use these roads to reach any city from the capital. But it is impossible to reach the capital from a city by roads.

    To solve this problem, there is a system of teleporters. Each city has a teleporter in it. Each teleporter has a value of strength. Let's say, the -th one has the value of strength equal to . This means that by using the teleporter, you'll become cities close to the capital (the same as going roads in the capital direction). If the distance to the capital is less than , you'll simply move to the capital instead.

    The road system of ByteLand and the initial teleports' strengths will be given to you. You'll also be given queries about the teleport network in ByteLand. These queries will have one of the following forms:

    • : The strength of the teleport in the city with the number becomes equal to ;

    • : Please output the minimal number of teleports that needs to be used to get from the city with the number to the capital. We can use only teleports in this query, but not the roads!

    Input Format

    The first line of input will contain two space-separated integers and .

    The second line will contain space separated integers - the initial strengths of the teleports.

    The following lines will contain pairs of space separated positive integers and with the meaning that there is a road between the cities with the numbers and .

    Each of following lines will contain a query in one of the formats, described above.

    Constraints

    1 <= N <= 100000
    1 <= M <= 100000
    1 <= Ai <= 10000

    Output Format

    For each query of the second type, output an answer on a separate line.

    In the first query, we jump two cities up and appear in the second city, then we jump once more and move to the capital.
